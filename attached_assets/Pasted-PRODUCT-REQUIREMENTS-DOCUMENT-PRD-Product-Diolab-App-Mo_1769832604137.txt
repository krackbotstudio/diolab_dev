PRODUCT REQUIREMENTS DOCUMENT (PRD)

Product: Diolab App
Module Name: OP POS
Feature Type: Core Module
Status: Approved
Priority: High

--------------------------------------------------
1. OVERVIEW
--------------------------------------------------
OP POS is a POS-style Out-Patient (OP) management module for Clinics and Hospitals.
It enables fast walk-in OP registration, online OP booking, queue management,
doctor consultation, and basic billing in a single operational interface.

--------------------------------------------------
2. PROBLEM STATEMENT
--------------------------------------------------
Clinics and hospitals rely on manual registers or slow, form-heavy software
to manage OP patients, leading to long queues, data errors, and poor
patient experience.

--------------------------------------------------
3. GOALS & OBJECTIVES
--------------------------------------------------
- Enable OP registration in under 30 seconds
- Provide a POS-like interface for reception staff
- Allow clinics to share public OP booking links
- Merge walk-in and online bookings into one queue
- Improve daily OP throughput

--------------------------------------------------
4. USER ROLES
--------------------------------------------------
1. Reception / Front Desk
2. Doctor
3. Clinic / Hospital Admin
4. Patient (External)

--------------------------------------------------
5. FEATURE SCOPE
--------------------------------------------------
IN SCOPE:
- OP POS dashboard
- Walk-in OP registration
- Online OP booking via clinic link
- Unified OP queue
- Doctor consultation panel
- Basic billing
- OP tracking for patients

OUT OF SCOPE (PHASE 1):
- Insurance & corporate billing
- In-patient management (IP)
- Advanced accounting
- Multi-branch hospital ERP

--------------------------------------------------
6. FUNCTIONAL REQUIREMENTS
--------------------------------------------------

6.1 OP POS DASHBOARD
- Live OP queue
- Auto token generation
- Status tracking:
  - Booked
  - Waiting
  - In Consultation
  - Completed
  - Cancelled
- Keyboard shortcuts for speed

6.2 WALK-IN OP REGISTRATION
- Minimal data entry:
  - Name
  - Mobile
  - Age / Gender
  - Department
  - Doctor (optional)
- New / Follow-up visit type
- Instant token assignment

6.3 ONLINE OP BOOKING
- Unique public booking link per clinic
- Patient booking without app install
- OTP verification (optional)
- Booking confirmation with token number

6.4 UNIFIED OP QUEUE
- Merge walk-in and online OPs
- Manual reorder option (admin)
- Color-coded status indicators

6.5 DOCTOR CONSULTATION PANEL
- View assigned OP list
- Patient details
- Symptoms & diagnosis
- Prescription entry
- Mark consultation completed

6.6 BILLING (POS STYLE)
- Default consultation fee
- Payment modes:
  - Cash
  - UPI
  - Unpaid
- One-click OP completion

6.7 PATIENT OP TRACKING
- Access via clinic booking link
- Mobile number + OTP
- Live queue status
- Visit history

--------------------------------------------------
7. USER FLOWS
--------------------------------------------------

WALK-IN OP FLOW:
New OP → Token Generated → Queue → Consultation → Billing → Completed

ONLINE OP FLOW:
Booking Link → OP Booking → Approved → Queue → Consultation → Completed

--------------------------------------------------

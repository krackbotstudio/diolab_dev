# Deploy Diolab on Render (Production Guide)

This repository now includes a **Render Blueprint** (`render.yaml`) so you can deploy the full stack in one flow (web service + PostgreSQL database).

## 1) Prerequisites

- A Render account
- Your code pushed to GitHub/GitLab
- API credentials you use in production (OpenAI, optional Google OAuth)

## 2) One-click-style deployment with Blueprint

1. In Render dashboard, click **New +** → **Blueprint**.
2. Connect your repository.
3. Render will detect `render.yaml` and propose:
   - `diolab-app` (Node web service)
   - `diolab-db` (PostgreSQL)
4. Click **Apply**.

Render will automatically:
- install dependencies (`npm ci`)
- build the app (`npm run build`)
- run production server (`npm run start`)

## 3) Required environment variables

Set these in Render (`diolab-app` → Environment):

### Required
- `SESSION_SECRET` (auto-generated by blueprint)
- `DATABASE_URL` (auto-injected from `diolab-db` by blueprint)

### Required for AI features
- `OPENAI_API_KEY`

### Optional (if using custom OpenAI-compatible base URL)
- `AI_INTEGRATIONS_OPENAI_BASE_URL`

### Optional (if using Google Sign-In)
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`

## 4) Database setup (important)

After first deploy, run schema migration against Render DB.

Use Render Shell on `diolab-app` and run:

```bash
npm run db:push
```

This applies your Drizzle schema to the production database.

## 5) Health check

Blueprint uses:

- `GET /api/health`

Expected response:

```json
{"status":"ok"}
```

## 6) Verify all major functionalities

After deployment, test these flows from the Render URL:

1. App loads and routes render correctly
2. Auth works (custom auth and Google auth if configured)
3. API endpoints return data (not 500)
4. Upload endpoints work
5. AI endpoints work (requires `OPENAI_API_KEY`)
6. WebSocket/signaling features connect

## 7) Common Render issues and fixes

### Build fails on dependency install
- Confirm Node version is 20+ (blueprint pins via `NODE_VERSION=20.19.0`).
- Redeploy after clearing build cache.

### App boots but DB queries fail
- Ensure `DATABASE_URL` is attached from Render DB service.
- Run `npm run db:push` once.

### Google login redirect mismatch
- Add your Render domain callback URL in Google Console.

### AI endpoints fail
- Set `OPENAI_API_KEY` in Render env vars and redeploy.

## 8) Recommended production hardening

- Keep `NODE_ENV=production`
- Rotate `SESSION_SECRET` periodically
- Move to persistent session store (Redis/Postgres) when scaling beyond single instance
- Enable custom domain + TLS on Render
